# Component: `src/lib/workshop-catalog.ts`

Purpose:
- Centralize workshop taxonomy decisions used across pages and scripts.

Inputs:
- `Workshop[]` and `Module[]` data structures from schemas.

Outputs:
- Canonical constants and helpers:
  - `CORE_WORKSHOP_IDS`
  - `isCoreWorkshop`
  - `isCoreWorkshopId` (alias)
  - category-aware core treatment for sessions that define `category`
  - `isVisualClassWorkshopId`
  - `splitWorkshops`
  - `isDisplayableVisualClassModule`

State:
- Pure in-memory helper set, no runtime side effects.

Edge cases:
- Workshop id normalization is case-insensitive by trimming/lowercasing incoming IDs.
- Visual-class module selection now gates only blocklist/title exclusions and no longer requires `module.visualClass`.
- The visual-class module blocklist prevents excluded modules from showing even if content still exists.

Dependencies:
- `src/lib/schemas.ts`.

Acceptance checks:
- All callers use `splitWorkshops()` for workshop grouping.
- Page-level grouping can include category-tagged workshops directly while sitemap generation still relies on `CORE_WORKSHOP_IDS`.
