# Component/page: `src/pages/workshops.astro`

Purpose:
- Render a workshops-only listing surface under `/workshops`.
- Present curated workshops grouped by category with lightweight OSAN polish.

Inputs:
- `loadWorkshops()` from `src/lib/content.ts`.
- `getCoreWorkshopCatalog()` and `getCoreWorkshopCatalogByCategory()` from `src/lib/learn-pages.ts`.

Outputs:
- Hero block with Learn/Workshops context.
- Workshops section with four ordered category shells and per-category card grids.
- Session count chips in each category header.

State:
- No client-side state.
- Build-time derivation and stable output via shared helper ordering.

Card behavior:
- Cards render:
  - `title`
  - `proof` (trimmed/clipped by helper)
  - one `cta` action button
- CTA precedence is delegated to shared helper logic (`Replay` when replay URL is valid, then `Slides`, else detail fallback).
- External links open in new tabs when `isExternal` is true.

Styling:
- Route-scoped category sections in `body[data-page='workshops']` receive muted structural polish, section headings, and subtle count pills.
- Existing `learn-card` interaction token behavior is preserved for consistency with the learn surfaces.

Dependencies:
- `CardGrid`
- `src/lib/content.ts`
- `src/lib/learn-pages.ts`

Edge cases:
- Missing proof text is normalized before render.
- Empty dataset or empty category renders a safe fallback message.
- Category metadata mismatch falls back to an "Other" section.

Acceptance checks:
- `/workshops` displays the provided titles/links grouped in the requested four categories.
- Categories and counts render in requested order.
- `/workshops/{id}` remains reachable via generated page links.
- `Replay` CTAs remain the external-first action for valid replay URLs.
