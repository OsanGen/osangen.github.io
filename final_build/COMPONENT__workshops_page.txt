# Component/page: `src/pages/workshops.astro`

Purpose:
- Render a workshops-only listing surface under `/workshops`.
- Preserve the reduced UI contract: title + proof snippet + single action per card.

Inputs:
- `loadWorkshops()` from `src/lib/content.ts`.
- `getCoreWorkshopCatalog()` from `src/lib/learn-pages.ts`.

Outputs:
- Hero block with eyebrow/title/subtitle.
- `Workshops` section with one card grid section and a canonical `coreWorkshops` list.

State:
- No client-side state.
- Build-time derivation and stable sorted output (`sortWorkshopsNewestFirst` in the helper).

Card behavior:
- Cards render:
  - `title`
  - `proof` (trimmed/clipped by helper)
  - `cta` action button
- CTA precedence is delegated to shared helper logic (Replay/Slides/Open details behavior where applicable).
- No date/meta chips or topic rows are rendered for workshop cards.

Dependencies:
- `CardGrid`
- `src/lib/content.ts`
- `src/lib/learn-pages.ts`

Edge cases:
- Missing proof text is normalized before render.
- Invalid CTAs receive fallback behavior from helper functions.
- Empty dataset renders a safe fallback message.

Acceptance checks:
- Only canonical workshop details appear in `/workshops`.
- `/workshops/{id}` remains reachable via generated page links.
- Card UI remains noise-reduced with one CTA and no extra helper rows.
