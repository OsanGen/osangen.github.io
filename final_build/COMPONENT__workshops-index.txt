# Component: `src/pages/workshops.astro` (workshops catalog surface)

Purpose:
- Provide the `/workshops` learn surface as a workshop-only listing.
- Keep card layout minimal while introducing category-first sectioning.

Inputs:
- `loadWorkshops()` from `src/lib/content.ts`.
- `getCoreWorkshopCatalog()` and `getCoreWorkshopCatalogByCategory()` from `src/lib/learn-pages.ts`.

Outputs:
- Hero block with workshop route context.
- Four section blocks under workshops, each with:
  - Category title
  - Session count chip
  - Card grid of canonical sessions for that category.

State:
- No client-side state.
- Build-time catalog transformation and deterministic output.

Card behavior:
- Canonical workshop set is produced via `getCoreWorkshopCatalog()` and grouped via `getCoreWorkshopCatalogByCategory()`.
- Cards render:
  - `title`
  - `proof`
  - one action button using `cta.label` / `cta.href`.
- CTA precedence remains from shared helpers (`Replay` > `Slides` > `Open details`).

Dependencies:
- `CardGrid`
- `src/lib/content.ts`
- `src/lib/learn-pages.ts`

Acceptance checks:
- Only canonical workshop cards render in this surface.
- The ordered category labels match the requested names.
- Empty categories render an explanatory fallback.
- `Replay`/`Slides` behavior remains unchanged for valid workshop links.
