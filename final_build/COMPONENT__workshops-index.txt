# Component: `src/pages/workshops.astro` (visual taxonomy surface)

Purpose:
- Provide a single Learn catalog route split into two deterministic blocks:
  - Core workshops (with canonical detail pages).
  - Visual classes (module cards plus non-core workshop cards).

Inputs:
- `loadModules()`, `loadWorkshops()`, `loadLastModifiedISO('workshops.json')`.
- `splitWorkshops()` and `isDisplayableVisualClassModule()` from `src/lib/workshop-catalog.ts`.

Outputs:
- Visual class list and workshop list sections with one action button per card.
- Last-updated timestamp for `workshops.json`.

State:
- No client-side state.
- All routing groups are computed at build time in page frontmatter.

Edge cases:
- Workshop entries with missing replay/slides render internal fallback CTA.
- Non-core workshops never render direct `/workshops/{id}` detail CTA in visual class section.

Dependencies:
- `Card`, `CardGrid`, `TagList`, `src/lib/link-utils.ts`.

Acceptance checks:
- Visual classes section includes non-core workshop-backed cards.
- Core workshop section contains only canonical IDs: `leanagents-workshop-chatgpt-apps`, `manus-workshop`.
