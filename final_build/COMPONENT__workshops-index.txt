# Component: `src/pages/workshops.astro` (workshops catalog surface)

Purpose:
- Provide the `/workshops` learn surface as a workshop-only listing.
- Keep card layout minimal: title, proof snippet, and one CTA.

Inputs:
- `loadWorkshops()` from `src/lib/content.ts`.
- `getCoreWorkshopCatalog()` from `src/lib/learn-pages.ts`.

Outputs:
- Hero block with route context (Learn/Workshops).
- Workshops section (`<section id="workshops">`) containing one card grid of canonical workshop entries.
- No quick-link rail, no card metadata chips, and no additional helper sections.

State:
- No client-side state.
- Build-time catalog transformation and deterministic sorting by date.

Card behavior:
- Canonical workshop set is produced by `splitWorkshops()` in `getCoreWorkshopCatalog`.
- Cards render:
  - `title`
  - `proof`
  - one action button using `cta.label` / `cta.href`.
- CTA precedence comes from shared helpers (`Replay` > `Slides` > `Open details`).

Edge cases:
- Missing links resolve to safe fallback route/actions.
- Empty workshop lists render a non-breaking copy fallback message.

Dependencies:
- `CardGrid`
- `src/lib/content.ts`
- `src/lib/learn-pages.ts`

Acceptance checks:
- Only canonical core workshop cards render in this surface.
- Cards do not include date text, proof badges, chip rows, or extra tag UI.
- `Replay`/`Slides`/`Open details` behavior remains unchanged for valid workshop links.
