# Component: `src/lib/learn-pages.ts`

Purpose:
- Centralize learn-surface transformations for module and workshop cards.
- Build deterministic curriculum sequencing, proof snippets, lesson metadata, and CTA selection in one place.

Inputs:
- `Module[]` and `Workshop[]` from `src/lib/schemas.ts`.
- Helper exports from `src/lib/workshop-catalog.ts` (`splitWorkshops`, `isDisplayableVisualClassModule`) and `src/lib/link-utils.ts`.

Outputs:
- `getVisualClassCatalog(modules: Module[])`: modules-only curriculum cards for `/visual-classes`.
- `getCoreWorkshopCatalog(workshops: Workshop[])`: canonical workshop cards for `/workshops`.
- `getWorkshopReplayLinks(workshops: Workshop[])` and `getWorkshopSlideLinks(workshops: Workshop[])`: valid quick links for future reuse.
- Learn card model (`LearnCard`) with:
  - `title`, `meta`, `proof`, `description`.
  - `lessonIndex` + `lessonLabel` for curriculum progression.
  - optional `slidesHref` + `slidesLabel`.
  - computed CTA state (`cta.href`, `cta.label`, `cta.isExternal`).
  - module slide-backed CTA labels use `Play` on `/visual-classes`.

Curriculum behavior:
- `VISUAL_CLASS_DECK_SEQUENCE` controls explicit learning order for modules.
- Current sequence is six lessons:
  - `vibe-to-pral`, `kinetic-kitchen-2-0`, `vpn-cost-lab`, `kitchen-x-ray`, `neural-glass`, `ai-kitchen`.
- `isDisplayableVisualClassModule()` removes blocked/non-approved module entries.
- Results are filtered by module id against the explicit sequence map before final sort, so removed modules are never rendered.
- Lesson labels are assigned from sequence position (`Lesson 1` ... `Lesson 6`) and remain stable even when copy changes.

Proof and continuity:
- `clampProof()` produces short proof lines for card surfaces.
- `withLearningProgress()` adds progression sentence text for non-proof details on catalog output.
- `withLearningProgressDescription()` similarly prefixes full descriptions with `Builds on <prior title>` when not already present.

CTA logic:
- Modules prefer `module.links.slides` via `buildModuleCta()`; when present, the module CTA label is `Play`. Fallback uses module-level CTA.
- Workshops prefer `replayUrl`, then `slidesUrl`, then detail routes for deterministic labels.
- `resolveVisualClassModuleSlideHref()` and `resolveSlideHref()` guard against unavailable links and internal fallback routes.

State:
- Pure functions only; no mutable global state.

Edge cases:
- Invalid or placeholder links are removed before surfacing in catalogs.
- Missing proof/description fallback to normalized safe defaults.
- Description and proof fields remain non-empty to prevent layout collapse.
- Empty catalog input returns an empty array with no runtime exception.

Dependencies:
- `src/lib/workshop-catalog.ts`
- `src/lib/link-utils.ts`
- `src/lib/schemas.ts`

Acceptance checks:
- `/visual-classes` renders exactly modules selected by schema + explicit sequence order.
- `/workshops` renders canonical workshop catalog from `splitWorkshops()`.
- Module and workshop CTA precedence is stable and label-safe.
- Module slide-backed CTAs render `Play`, while workshop CTA labels remain unchanged (`Replay`/`Slides`/`Open details`).
- Lesson count and labels are consistent with the page hero text (`Lesson 1 of 6`).
