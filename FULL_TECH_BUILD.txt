## 2) What this repo is
This is an Astro-based OSAN website for portfolio, workshops, and learning modules with proof-oriented cards, motion accents, and route-specific visual treatment.

### User-facing artifacts
- Public routes: `src/pages/**` (notably `/workshops`, `/visual-classes`, `/labs`, `/join`, `/gpts`, `/resume`, `/more`).
- Shared app chrome and flow composition are in `src/layouts/BaseLayout.astro`, `src/components/Header.astro`, and `src/components/Footer.astro`.
- Content is JSON-driven with schema validation and typed catalog helpers.

## 3) System map
- Shell:
  - `src/layouts/BaseLayout.astro`: route wrapper, metadata, and visual tokens.
  - `src/components/Header.astro`, `src/components/Footer.astro`: nav and brand consistency.
  - `src/components/MotionEnhance.astro` and `src/scripts/osan-motion.ts`: motion orchestration and reduced-motion handling.
  - `src/components/ServiceWorkerRegister.astro` + `public/sw.js`: cache and offline baseline.
- Content layer:
  - `src/content/data/*.json` content payloads.
  - `src/lib/content.ts`, `src/lib/schemas.ts`: loading + validation contracts.
  - `src/lib/learn-pages.ts`: learn helpers and card mapping.
  - `src/lib/workshop-catalog.ts`: workshop classification + core/non-core split behavior.
- Styling:
  - `src/styles/tokens.css`, `src/styles/global.css`, and page-level styles.
- Validation/ops:
  - `scripts/validate-content.ts`
  - `scripts/check-dist-integrity.mjs`
  - `scripts/runtime-http-smoke.mjs`

## 4) Entrypoints and run commands

- Runtime smoke: `npm run verify:runtime`

- Environment notes:

## 6) Current implementation status
- What is implemented:
  - `/resume` hero is now labeled `About Me` and uses creator-focused headline copy (`Creator of this site, games, workshops, and modules.`).
  - `/resume` skills now use a technical, engineer-facing taxonomy aligned to delivered work (agentic systems, Astro/TS architecture, CI/CD, service worker reliability, and educational game development).
  - `/resume` education cards now display only program and institution; the date subline under each education item is removed.
  - Shared primary nav label for `/resume` is now `About Me` across header and footer tabs via `src/lib/site-nav.ts`.
  - `/visual-classes` lesson cards now omit duration/level meta rows (`90 minutes • Intermediate`) so the carousel is lesson-title and action focused.
  - Deploy hotfix applied in `src/lib/learn-pages.ts`: `LearnCard` now includes optional `category`, matching grouped workshop rendering usage and removing CI `astro check` failures.
  - Workshop dataset replaced with curated events grouped into:
    - `Gen AI Live (Guests and Episodes)`
    - `Agents and Workflow Automation`
    - `Visual Media Creation (Image + Video)`
    - `Model and Platform Workshops`
  - Each workshop includes `category`, deterministic `date`, proof metadata, and action URLs.
  - `Workshop` and `LearnCard` schemas/types support `category`.
  - `getCoreWorkshopCatalog` retains core behavior while `getCoreWorkshopCatalogByCategory` powers grouped sections.
  - `/workshops` renders category sections with counts and accent-subtle OSAN style polish.
  - `isCoreWorkshop` now treats categorized rows as core entries, with legacy `CORE_WORKSHOP_IDS` fallback preserved.
  - Homepage workshops card count now uses the same curated helper list as `/workshops`.
  - Related component docs and current-gap tracker updated.
- What is in progress:
  - Manual post-deploy verification on `/workshops` and mobile responsive behavior.
- What is not started:
  - Dedicated automated tests around new category helper output ordering and fallback grouping.
